# 🔌 مرجع واجهات برمجة التطبيقات - NGLP API Reference (باللغة العربية)

يحتوي هذا المستند على التفاصيل الكاملة والدليل التشغيلي لمسارات الـ **API** المدعومة في منصة **NGLP Backend**. 
العنوان الأساسي (Base URL) الافتراضي للتشغيل المحلي هو: **`http://localhost:8080`**.

---

## 👥 1. مسارات المستخدمين والمصادقة (Users & Authentication)
التحكم بحسابات المستخدمين وصلاحيات تسجيل الدخول والتسجيل الجديد.

### تسجيل حساب جديد (Register)
* **المسار**: `POST /api/auth/register`
* **الوصف**: إنشاء حساب جديد لطالب أو معلم في النظام.
* **الترويسات (Headers)**: `Content-Type: application/json`
* **مثال لمدخلات الطلب (Request Body)**:
```json
{
  "fullName": "أحمد الطالب",
  "email": "student@nglp.com",
  "password": "student123",
  "role": {
    "id": 1,
    "name": "ROLE_STUDENT"
  }
}
```
* **الاستجابة الناجحة (200 OK)**: تُرجع بيانات المستخدم الذي تم إنشاؤه بنجاح (مع إخفاء كلمة المرور لدواعي الأمان).

### تسجيل الدخول (Login)
* **المسار**: `POST /api/auth/login`
* **الوصف**: التحقق من البريد الإلكتروني وكلمة المرور والحصول على جلسة مصادقة.
* **مثال لمدخلات الطلب**:
```json
{
  "email": "student@nglp.com",
  "password": "student123"
}
```
* **الاستجابة الناجحة (200 OK)**:
```json
{
  "id": 2,
  "fullName": "أحمد الطالب",
  "email": "student@nglp.com",
  "role": {
    "id": 1,
    "name": "ROLE_STUDENT",
    "description": "Student learner account"
  }
}
```

### عرض جميع المستخدمين (Get All Users)
* **المسار**: `GET /api/users`
* **الوصف**: جلب قائمة بجميع المستخدمين المسجلين في النظام (خاص بالإدارة والمعلمين).
* **الاستجابة الناجحة (200 OK)**: مصفوفة تحتوي على كائنات المستخدمين.

### جلب مستخدم محدد بالمعرف (Get User by ID)
* **المسار**: `GET /api/users/{id}`
* **المتغيرات في الرابط (Path Variables)**: `{id}` هو الرقم التعريفي للمستخدم (مثال: `2`).
* **الاستجابة الناجحة (200 OK)**: بيانات المستخدم المطلوب فقط.

### حذف مستخدم (Delete User)
* **المسار**: `DELETE /api/users/{id}`
* **الوصف**: حذف حساب مستخدم نهائياً من النظام عبر معرفه الرقمي.
* **الاستجابة الناجحة (200 OK)**: استجابة فارغة تؤكد إتمام الحذف.

## 🎓 2. مسارات الكورسات والدروس (Courses & Lessons)
إدارة المحتوى التعليمي والكورسات والتصنيفات.

### عرض جميع الكورسات (Get All Courses)
* **المسار**: `GET /api/courses`
* **الوصف**: جلب جميع الدورات التعليمية المتاحة في المنصة.
* **الاستجابة**: مصفوفة تحتوي على تفاصيل الكورسات والتصنيف والمدرب المرتبط بكل كورس.

### إنشاء كورس جديد (Create Course)
* **المسار**: `POST /api/courses`
* **الوصف**: إضافة دورة تعليمية جديدة بواسطة معلم أو مسؤول النظام.
* **مثال لمدخلات الطلب**:
```json
{
  "title": "تعلم أساسيات الويب",
  "description": "دورة لتعليم أساسيات HTML و CSS للمبتدئين",
  "category": {
    "id": 1
  }
}
```

### عرض الدروس التابعة لكورس (Get Lessons)
* **المسار**: `GET /api/lessons`
* **الوصف**: عرض كافة الدروس البرمجية الموجودة في النظام.

### إضافة درس جديد إلى كورس (Create Lesson)
* **المسار**: `POST /api/lessons`
* **الوصف**: ربط درس جديد بكورس محدد وتحديد مسار الفيديو الخاص به.
* **مثال لمدخلات الطلب**:
```json
{
  "title": "الدرس الأول: بنية صفحة الويب HTML",
  "videoUrl": "uploads/videos/html_lesson1.mp4",
  "durationSeconds": 600,
  "course": {
    "id": 1
  }
}
```

## 📝 3. مسارات تفريغ النصوص والتحاق الطلاب (Transcripts & Enrollments)

### تسجيل طالب في كورس (Enroll Student in Course)
* **المسار**: `POST /api/enrollments`
* **الوصف**: التحاق طالب بكورس تعليمي لبدء المتابعة وتفعيل المعلم الذكي.
* **مثال لمدخلات الطلب**:
```json
{
  "userId": 2,
  "courseId": 1
}
```

### تحديث تقدم الطالب (Update Progress)
* **المسار**: `PUT /api/enrollments/{id}/progress`
* **الوصف**: تحديث النسبة المئوية لتقدم الطالب في الكورس والدرس الأخير الذي شاهده.
* **مثال لمدخلات الطلب**:
```json
{
  "progressPercentage": 45,
  "lastWatchedLessonId": 3
}
```

### جلب نصوص تفريغ الدرس (Get Transcripts by Lesson)
* **المسار**: `GET /api/lessons/{lessonId}/transcripts`
* **الوصف**: جلب كل القطع النصية المفرغة والمرتبطة بالزمن لدرس معين. يستخدمها النظام والـ AI لمعرفة محتوى شرح الفيديو.

## 🤖 4. مسار المعلم الذكي (AI Tutor Endpoint)
نقطة الاتصال الرئيسية للتخاطب مع مساعد الذكاء الاصطناعي أثناء الدرس.

### طرح سؤال على المعلم الذكي (Ask AI Tutor)
* **المسار**: `POST /api/ai/ask`
* **الوصف**: إرسال سؤال من الطالب حول جزئية معينة من الدرس. سيقوم النظام تلقائياً بجلب تفريغ الدرس وإرساله كخلفية معرفية لنموذج LLaMA ليجيب بناءً عليها مع تذكر المحادثات السابقة.
* **الترويسات (Headers)**: `Content-Type: application/json`
* **مثال لمدخلات الطلب (Request Body)**:
```json
{
  "userId": 2,
  "lessonId": 1,
  "timestamp": "00:45",
  "message": "ماذا تعني ميزة الـ Components المذكورة في هذا الوقت من الدرس؟",
  "conversationId": "1a2b3c4d-5e6f-7g8h"
}
```
* **شرح معلمات الطلب (Request Parameters)**:
  - `userId`: معرف الطالب السائل لتوثيق وحفظ الرسالة باسمه.
  - `lessonId`: معرف الدرس الحالي لكي يقوم الذكاء الاصطناعي بقراءة تفريغ هذا الدرس تحديداً.
  - `timestamp`: التوقيت الزمني بالدقيقة والثانية الذي يطرح الطالب سؤاله عنده في الفيديو (مثال: `00:45`).
  - `message`: نص سؤال الطالب.
  - `conversationId` *(اختياري)*: معرف المحادثة الحالية للاستمرار في نفس السياق وتذكر الأسئلة السابقة. إذا لم يُرسل، سيقوم النظام بإنشاء معرف محادثة جديد وإرجاعه في الاستجابة.

* **مثال للاستجابة الناجحة (200 OK)**:
```json
{
  "conversationId": "1a2b3c4d-5e6f-7g8h",
  "answer": "أهلاً بك! في التوقيت (00:45) من الفيديو، يتحدث المحاضر عن 'المكونات' (Components) في React. المكونات هي بمثابة قطع بناء مستقلة وقابلة لإعادة الاستخدام لتكوين واجهة المستخدم، تماماً مثل قطع الـ Lego. هذا يساعد في تنظيم الكود وعدم تكرار كتابته."
}
```

## 💬 5. مسارات المحادثات (Conversations)

### عرض محادثات المستخدم (Get User Conversations)
* **المسار**: `GET /api/conversations/user/{userId}`
* **الوصف**: جلب جميع جلسات المحادثة التاريخية التي أجراها الطالب مع الـ AI Tutor.

### جلب رسائل محادثة معينة (Get Messages in Conversation)
* **المسار**: `GET /api/conversations/{conversationId}/messages`
* **الوصف**: جلب قائمة بكافة الرسائل (السؤال والجواب) المتبادلة في محادثة معينة لعرضها في شات واجهة المستخدم.

---

## 💡 نصائح سريعة للمطورين المبتدئين
1. **تحديد نوع البيانات (Content-Type)**: تذكر دائماً وضع رأس `Content-Type: application/json` في البرمجيات أو في Postman لجميع طلبات الإرسال (POST/PUT).
2. **التعامل مع المتغيرات (Path Variables)**: في المسارات التي تحتوي على نقطتين أو أقواس متعرجة مثل `:id` أو `{id}`، استبدلها بالرقم الفعلي للمستخدم أو الكورس قبل إرسال الطلب (مثال: `/api/users/2`).
3. **فهم مخرجات الـ AI**: عند استخدام `/api/ai/ask` سيقوم السيرفر بحفظ المحادثة تلقائياً في قاعدة بيانات H2، مما يعني أنه يمكنك فوراً عمل استدعاء لـ GET `/api/conversations/{id}/messages` لمشاهدة التحديث الفوري للرسائل.
