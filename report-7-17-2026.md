## 5. Database Schema & Entity Relationships

```
Role (1) ──── (N) User
Category (1) ──── (N) Course  (self-referencing parent/child)
User/Teacher (1) ──── (N) Course
Course (1) ──── (N) Lesson
Lesson (1) ──── (N) LessonTranscript
Lesson (1) ──── (N) Conversation
User (1) ──── (N) Conversation
Conversation (1) ──── (N) Msg
User (1) ──── (N) Enrollment
Course (1) ──── (N) Enrollment
```

### Entity Details

**User** (`users` table): id, fullName, email (unique), password (BCrypt), blocked (Boolean), role (FK→Role)

**Role** (`roles` table): id, name (e.g. ROLE_STUDENT), description

**Category** (`categories` table, self-referencing): id, name, parent (FK→Category), subCategories

**Course** (`courses` table): id, title, description (TEXT), category (FK), teacher (FK→User)

**Lesson** (`lessons` table): id, title, description (TEXT), videoUrl, durationSeconds, course (FK), transcripts (cascade ALL), conversations (cascade ALL)

**LessonTranscript** (`lesson_transcripts` table): id, lesson (FK), startSecond, endSecond, transcriptContent (TEXT)

**Enrollment** (`enrollments` table): id, user (FK), course (FK), progressPercentage (0-100), lastWatchedLesson (FK→Lesson), unique(user_id, course_id)

**Conversation** (`conversations` table): id, user (FK), lesson (FK), startedAt (LocalDateTime)

**Msg** (`messages` table): id, conversation (FK), senderType ("USER"/"ASSISTANT"), videoTimestamp, content (TEXT), sentAt (LocalDateTime)

## 6. Services / Modules

### Service Classes (9)
1. **UserService** - CRUD, auth, BCrypt encoding, admin updates, delete
2. **RoleService** - Find all/by ID/by name
3. **CourseService** - CRUD, filtering by category/teacher, safe delete
4. **LessonService** - CRUD, create with video upload → async transcription
5. **CategoryService** - CRUD, root/subcategory hierarchy, safe delete
6. **EnrollmentService** - Enroll (idempotent), progress (clamped 0-100), list by user
7. **ConversationService** - Get or create conversation (one per user-lesson pair), chat history, delete
8. **LessonTranscriptService** - Async transcription via Python server, find by lesson/timestamp, timestamp parsing ("MM:SS" or raw seconds)
9. **FileStorageService** - UUID-based video saving, path sanitization

### AI Layer (3 files)
1. **NglpAiAgent** - Core AI orchestrator; ChatClient config with pedagogical system prompt; synchronous & streaming methods; injects transcript context (RAG)
2. **DatabaseChatMemory** - Implements Spring AI `ChatMemory`; persists messages to DB; retrieves last N (default 8) messages; strips injected context from stored messages
3. **AiToolsConfig** - `@Tool` annotation for `fetchLessonTranscript` function calling

### Supporting
- **DataInitializer** - Seeds 3 roles, 3 users, 4 categories, 3 courses, 5 lessons with transcripts, 3 enrollments; copies demo videos
- **WebConfig** - Static resource serving from `uploads/`
- **ResourceNotFoundException** - Custom runtime exception

## 7. Security

### Authentication
- **Custom header-based auth** (`X-User-Role` header) - NO JWT, NO OAuth2, NO sessions
- Stateless session creation policy

### Authorization
- Public: register, login, read endpoints (courses, categories, roles), H2 console, Swagger
- Admin-protected: checked manually in controllers via `X-User-Role` header
- All others: `.anyRequest().permitAll()` (effectively open)

### Security Features
- CSRF disabled
- CORS: `http://localhost:5173`, `http://127.0.0.1:5173` (Vite dev server)
- Frame options: `sameOrigin` (H2 console)
- BCryptPasswordEncoder configured

## 13. Notable Observations & Issues

1. **API key hardcoded** in `application.properties` - security risk
2. **Security is effectively open** - `.anyRequest().permitAll()`, header-based admin check is trivial to bypass
3. **No JWT/token auth** - `AuthResponse.token` field exists but is never populated
4. **Doc vs reality mismatch** - PROJECT_REPORT.md mentions MySQL/OpenAI/Groq but code uses H2/Gemini
5. **MsgService missing** - referenced in docs but not implemented
6. **No LessonTranscriptController** - referenced in architecture diagram but absent
7. **No notification system** - MsgRepo has `deleteByConversationId` but no notification infrastructure
8. **No global exception handler** - no `@ControllerAdvice`
9. **Misleading TODO** - `password //TODO ENCRYPTION` comment exists but BCrypt is already applied
10. **Minimal test coverage** - only context load test
11. **External Python server dependency** - transcription requires Python server at port 8000
12. **Hardcoded paths** - upload dir, AI service URL
13. **Hard delete instead of soft delete** - comment recommends soft delete but actual delete is hard
14. **IntelliJ IDEA project** - `.idea/` directory present

## 16. Summary Statement

NGLP Backend v1 is a Java 20 / Spring Boot 4.0.6 monolith with an MVC-layered structure. It serves as an AI-powered educational platform with video lesson management, timestamped transcripts, AI tutoring via Google Gemini (with RAG from lesson transcripts), conversation history, enrollment tracking, and multi-role user management. The application is in an early MVP stage using H2 for persistence, has minimal security, no real test coverage, and has several documentation-reality gaps. It depends on an external Python server for video transcription.
